## 1. このシステムは何か（一言で）

> **「指示を出すと、決まった品質の画像素材が、決まったルールで量産される仕組み」** を作る。

ComfyUIを手で触る代わりに、**Manager（司令塔）**に「立ち絵のサクラ、表情差分ぜんぶ」と頼めば、
適切な**スタイル別agent**が選ばれ、**workflow**と**プロンプト**が組まれ、**一括生成**まで走る ── という状態がゴール。

さらに、自分以外（**他のAI/エージェント**）からも依頼が来ても、**ルールとスキーマ**さえ守れば
安全に受け付けられるようにする。

## 5. 環境の前提（毎回の鉄則）

これは「絶対に外せない運用ルール」。詳細は環境レポート参照。

1. **起動は必ず `Run_CPU.bat` 経由**（このCPU機）。CMD直叩きは `torch` が見つからずエラー。
   - GPU機で本生成する時は `--cpu` を外した起動（将来 `Run_GPU.bat` を用意予定）。
2. **本生成はGPU機**。このPCはGPUなし（CUDA=False）なので試作・確認用。
3. **一括生成スクリプトはComfyUIを先に起動してから**（`http://127.0.0.1:8188` が開くこと）。
4. ⚠️ **`transformers 5.0.0` 問題**: Qwen系等でエラーが出たら4系へ戻す（手順は環境レポート§4）。
5. 生成物は `D:\ai\ComfyUI\output\` に出る（用途別サブフォルダあり）。

### `Run_CPU.bat` の正体（依頼書の疑問への回答）
- `Run_CPU.bat` は **CPUモード起動BAT**（`python main.py --cpu`）。グラフィックボードを使わない方。
- 「GPUあり/なし両方作った」記憶のうち、**GPUなし＝この `Run_CPU.bat`**。GPU版BATは現在消失。
- 必要なら `Run_GPU.bat`（`--cpu`なし）を新規作成して2本並べると今後迷わない。

## 6. 「立ち絵・差分agent」を最初に作る理由

- 既に **土台が完成している**から（フェーズ0で発見）:
  - `generate_variants.py` … API経由で差分を一括生成するエンジン
  - `character_variations.json` … キャラ3・表情18・ポーズ16・背景8・優先25組の定義
  - 操作マニュアル(`0304_01_python_lecture.md`)とテスト用WF4種
- つまり「立ち絵・差分」は**ゼロからではなく、整理・標準化するだけ**で最初のagentにできる。
- ここで「依頼→定義→生成→出力」の1サイクルを完成させ、**他スタイルagentの雛形**にする。

## 7. 用語集（迷ったら）

| 用語 | 意味 |
| ---- | ---- |
| Manager | 依頼を受けて担当agentに振り分ける司令塔。人間にもAIにも窓口は1つ |
| agent | 特定ジャンルの専門担当（立ち絵/ピクセル/水彩…）。指示書の集合体 |
| workflow | ComfyUIのノードグラフ(.json)。実際の生成処理 |
| 差分 | 同一キャラの表情・ポーズ違いのバリエーション |
| greenback | 緑背景で生成し、後で背景を抜く立ち絵方式 |
| CG（このプロジェクトでの意味） | 背景込みの一枚絵（立ち絵＋シーン背景合成） |
| i2i | image to image。参照画像をもとに生成 |
| スキーマ | 依頼データの「型」。これに従えばAIでも安全に依頼できる |
| LoRA | モデルに味付けする追加学習データ（スタイル/キャラ） |
