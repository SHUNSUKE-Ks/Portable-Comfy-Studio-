## ■ 判明事項（重要）

- **グラボなしCPU機でComfyUI起動は可能**（実証済み）。**GPUの有無は障害ではない**。
- ✅ **2026-05-30 実機確認: `ComfyUI_CPU_RUN.bat`(venv, `--cpu`)でサーバ起動成功**。約40秒でready、Web UI が HTTP 200 応答、`/system_stats` 正常（v0.11.1 / python3.10.11 / pytorch2.5.1+cu121 / embedded_python:false=venv）。
- **実起動での import失敗は1件だけ**：`ComfyUI_Jags_Audiotools` = `No module named 'pandas'`（venv未導入）。`pip install pandas` で解消可。
- ⚠️ **訂正**: 以前 `comfyui_test_validate.py` が報告した「`utils` 名前衝突6件」は、**検証スクリプトのimport方法が生んだ誤検知**だった。実際の `main.py --cpu` 起動では `utils` 衝突は**発生しない**（実起動ログで0件確認）。→ 検証スクリプトのパッケージ検証部は実起動経路（`--quick-test-for-ci`等）に合わせて要修正。
- 当初疑った `transformers 5.0.0` も原因ではなかった。**結論: 環境はほぼ健全。実質 `pandas` 1個だけ不足**。
- 注: 起動時 空きRAMが約0.9GB/16GBまで低下。XL等の重いモデルをCPUで動かす場合はRAMがタイトな点に留意。
- **greenbackワークフロー**（`0304_01_greenback_*`）は使用ノード9種すべて**コア**＋モデル2個（`novaOrangeXL_v140.safetensors`, `granblue-klein9b.safetensors`）のみ。**カスタムノード不要＝上記失敗と無関係**。GPU機でそのまま生成可能。
