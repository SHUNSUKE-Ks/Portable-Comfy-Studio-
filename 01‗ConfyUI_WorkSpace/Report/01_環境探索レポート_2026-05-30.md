## 4. 🔴 安定性リスクと対処（最重要セクション）

### リスク①: transformers 5.0.0（メジャー更新）
- **症状の出方**: ComfyUI起動自体は通ることが多いが、`ComfyUI-qwenmultiangle`・`comfyui_controlnet_aux`・一部のテキストエンコーダ系で `ImportError` / `AttributeError` が出る可能性。
- **なぜ起きた**: 別作業や別ノードのインストールで `pip install` が走り、transformersが5系に巻き上げられたと推測。
- **対処方針（再稼働後にエラーが出てから対応でOK）**:
  1. まず現状のまま起動 → ログを確認（エラーが出なければ放置でよい）。
  2. Qwen系等でエラーが出たら、4系へダウングレード:
     ```bat
     D:\ai\ComfyUI\venv\Scripts\activate.bat
     pip install "transformers>=4.50,<5.0"
     ```
  3. ダウングレード前に現状を控える（ロールバック用）:
     ```bat
     pip freeze > D:\ai\01‗ConfyUI_WorkSpace\Report\pip_freeze_2026-05-30_backup.txt
     ```

> ⚠️ **今この場で勝手に更新・ダウングレードはしていません。** あなたの環境を壊さないため、判断はあなたに委ねます。実行する場合は上記コマンドを使ってください。

### リスク②: numpy 2.x
- 多くは問題ないが、古いノードで `np.float`/`np.int` 廃止由来のエラーが出ることがある。
- 対処はリスク①と同様、エラーが出たノード単位で対応。

### リスク③: 起動はBAT必須（既知）
- `torch` はvenv内のみ。CMD直叩きは `ModuleNotFoundError`。必ずBAT経由。
